# 4.7.0

Release date: 8 May 2026

:::tip

By default, the Visual Studio Code based editor will automatically update the MetaEd extension for you. If you are still using the deprecated Atom based editor, then please see [MetaEd IDE - Upgrading to MetaEd 4.x](../ide-user-guide/upgrading-to-metaed-4x.md).

:::

## Overview

This release supports version 6.1 of the Ed-Fi Data Standard, and Ed-Fi ODS/API
version 7.3.

## Changes

### Enhancements

* Added Support for Data Standard 6.1.
* Modernized the HTML in the HTML handbook, resulting in smaller output files.
* New [API Catalog spreadsheet](../ide-user-guide/using-the-generated-artifacts.md#api-catalog) listing all resource names and JSON attributes as they will appear in the Open API specification. [Sample spreadsheet](https://github.com/user-attachments/files/26647178/Ed-Fi-API-Catalog_2026-04-11.xlsx) (link to GitHub).

### Removals

* Removed support for Data Standard 3.2c and 3.3b.

## Bug Fixes

* Validation of domain entities in subdomains for Data Standard version 6+. Previously, a domain entity could be deleted from the model without causing a validation error in a subdomain.
* Validation that a Common's properties do not have the same as an identity property on the parent. Because this could cause a breaking change, it only applies for Data Standard 6.1 and later.
* Some elements in the HTML data handbook did not include "used by" information.
