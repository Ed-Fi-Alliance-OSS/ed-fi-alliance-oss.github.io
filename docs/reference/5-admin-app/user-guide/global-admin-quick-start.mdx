# Global Admin Quick Start

After installing the Ed-Fi Admin App, a global administrator normally has to
click through the UI to create a user, a team, an environment, and the ownership
that ties them together before reaching the screens that matter — environment
and instance management. This page provides a SQL script that stands up that
whole starter configuration so an administrator can sign in and go straight to
managing ODS instances.

:::tip Prefer a fully connected environment?

This SQL script seeds the data offline, so live Admin API operations
(Applications, Profiles, Ed-Org sync) do not authenticate until the environment
is connected. For a working, connected environment created through the API
instead, use the
[Global Admin Quick Start (API)](./global-admin-quick-start-api).

:::

The script configures, in the **Admin App application database**:

- An administrative user with the global **Global admin** role
- A single **team**, with the user as a **member**
- A single **environment** (default name `Ed-Fi ODS/API v7.3`, label
  `QuickStart`) with an injected `configPublic`
- A **default** Ed-Fi tenant
- One **ODS instance** per row in the seed list (`EdFi_Ods_2026`,
  `EdFi_Ods_2027`), each with one **education organization**
- **Ownership** of both the environment and the tenant by the team

:::warning Read this before running

The script **injects a `configPublic` JSON literal** directly into the
environment row. Normally an environment is created through the UI's **Connect
new environment** flow, which contacts the Admin API and ODS/API to detect the
version, register OAuth credentials (stored encrypted in `configPrivate`), and
sync ODS instances and education organizations.

Because of that, two things still depend on values this script cannot generate:

- **Admin API calls (Applications, and Ed-Org sync) need credentials.** The
  injected `configPublic` leaves the tenant's `adminApiKey` empty and
  `configPrivate` `NULL`, so those operations will return an authentication
  error. To enable them, set the `adminApiKey` (in `configPublic`) and the
  `adminApiSecret` (in the `configPrivate` variable) for the tenant. (The ODS
  instances and Ed-Orgs _lists_ read the local rows this script seeds, so they
  work without credentials.)
- **Live data still requires the ODS/API side.** The script writes local ODS
  instance rows so the ODS page lists them, but the ODS/API only serves data
  once `dbo.OdsInstances` in `EdFi_Admin` is populated (see
  [Step 2](#step-2-set-up-ods-instances-in-edfi_admin)).

Edit the `configPublic` (and the optional `configPrivate`) to match your
deployment before running.

:::

:::info Which database?

This script runs against the **Admin App application database** (the database
the Admin App API uses for its own state; named `sbaa` in the reference
deployment). It is _not_ the `EdFi_Admin` or `EdFi_Security` database.

:::

## Prerequisites

- The Ed-Fi ODS/API and Ed-Fi ODS Admin API are already installed and reachable
  at the URLs in the injected `configPublic`.
- The Admin App database has been created and its migrations have run.
  Migrations seed the built-in roles this script references (for example,
  `Global admin`).
- An **identity provider (OIDC)** is configured and the administrator can
  authenticate against it. This script makes the user _authorizable_, not
  _authenticatable_ — sign-in still depends on the IdP. See
  [Configuring an Identity Provider for Ed-Fi Admin App](/reference/admin-app/configuration/identity-provider).
- The username you use below **matches the subject the IdP will return**, and
  should match the `ADMIN_USERNAME` used when the Admin App database was
  bootstrapped.

## Variables

Set these at the top of the script before running it. They double as **lookup
keys**, which makes the script safe to re-run — existing rows are reused (and
the environment's `configPublic` / `envLabel` are refreshed) rather than
duplicated.

| Variable           | Default                          | Description                                                                   |
| ------------------ | -------------------------------- | ----------------------------------------------------------------------------- |
| `username`         | `admin@example.org`              | Administrator's username; must match the OIDC subject / `ADMIN_USERNAME`.     |
| `team name`        | `Quick Start`                    | Name of the single team to create.                                            |
| `environment name` | `Ed-Fi ODS/API v7.3`             | Display name of the environment.                                              |
| `env label`        | `QuickStart`                     | Stored in `sb_environment.envLabel`.                                          |
| `edfi tenant`      | `default`                        | Tenant name under the environment.                                            |
| ODS seed list      | `EdFi_Ods_2026`, `EdFi_Ods_2027` | One ODS instance + one Ed-Org per row; edit or extend the list in the script. |
| `configPublic`     | _(provided literal)_             | Injected into the environment; edit URLs / `adminApiKey` to match your site.  |
| `configPrivate`    | `NULL`                           | Optional — set to `{"tenants":{"default":{"adminApiSecret":"..."}}}` to auth. |

## Step 1: Run the quick-start script

Edit the variables (and the ODS seed list) at the top, then run the script
against the Admin App database with your preferred SQL client.

<Tabs groupId="db-engine">
<TabItem value="pgsql" label="PostgreSQL" default>

```sql
DO $$
DECLARE
  -- ---- Edit these variables ----
  v_username    text := 'admin@example.org';   -- = ADMIN_USERNAME / OIDC subject
  v_team_name   text := 'Quick Start';
  v_env_name    text := 'Ed-Fi ODS/API v7.3';
  v_env_label   text := 'QuickStart';
  v_edfi_tenant text := 'default';
  v_edorg_discriminator text := 'edfi.Other';
  -- configPublic to inject (single-quoted; edit URLs / adminApiKey to match your site)
  v_config_public jsonb := '{"startingBlocks":false,"odsApiMeta":{"version":"7.3","applicationName":"Ed-Fi Alliance ODS/API","informationalVersion":"7.3.2","suite":"3","build":"7.3.20305.0","dataModels":[{"name":"Ed-Fi","version":"6.1.0","informationalVersion":"The Ed-Fi Data Model 6.1"}],"urls":{"dependencies":"https://localhost/WebApi/metadata/data/v3/dependencies","openApiMetadata":"https://localhost/WebApi/metadata/","oauth":"https://localhost/WebApi/oauth/token","oauthTokenIntrospection":"https://localhost/WebApi/oauth/token_info","dataManagementApi":"https://localhost/WebApi/data/v3/","xsdMetadata":"https://localhost/WebApi/metadata/xsd","changeQueries":"https://localhost/WebApi/changeQueries/v1/","composites":"https://localhost/WebApi/composites/v1/"}},"adminApiUrl":"https://localhost/AdminApi","version":"v2","values":{"meta":{"envlabel":"Development","mode":"SingleTenant","domainName":"https://localhost/WebApi","adminApiUrl":"https://localhost/AdminApi","tenantManagementFunctionArn":"","tenantResourceTreeFunctionArn":"","odsManagementFunctionArn":"","edorgManagementFunctionArn":"","dataFreshnessFunctionArn":""},"adminApiUuid":"0f1e4a03-b5cd-4af4-a84a-5df7b21579e8","tenants":{"default":{"adminApiKey":""}}}}';
  -- Optional: set the encrypted secret so Admin API calls can authenticate.
  v_config_private jsonb := NULL;  -- e.g. '{"tenants":{"default":{"adminApiSecret":"<secret>"}}}'
  -- ---- Built-in role ids (seeded by migrations; do not change) ----
  c_role_global_admin   constant int := 2;  -- "Global admin"   (UserGlobal)
  c_role_tenant_admin   constant int := 6;  -- "Tenant admin"   (UserTenant)
  c_role_full_ownership constant int := 5;  -- "Full ownership" (ResourceOwnership)
  -- ----
  v_user_id   int;
  v_team_id   int;
  v_sbenv_id  int;
  v_tenant_id int;
  v_ods_id    int;
  v_edorg_id  int;
  r           record;
BEGIN
  -- Section 1: user, team, membership
  SELECT id INTO v_user_id FROM "user" WHERE username = v_username; -- citext: case-insensitive
  IF v_user_id IS NULL THEN
    INSERT INTO "user" (username, "roleId", "isActive", "userType")
    VALUES (v_username, c_role_global_admin, true, 'human') RETURNING id INTO v_user_id;
  ELSE
    UPDATE "user" SET "roleId" = COALESCE("roleId", c_role_global_admin), "isActive" = true
     WHERE id = v_user_id;
  END IF;

  SELECT id INTO v_team_id FROM team WHERE name = v_team_name;
  IF v_team_id IS NULL THEN
    INSERT INTO team (name) VALUES (v_team_name) RETURNING id INTO v_team_id;
  END IF;

  IF NOT EXISTS (SELECT 1 FROM user_team_membership WHERE "teamId" = v_team_id AND "userId" = v_user_id) THEN
    INSERT INTO user_team_membership ("teamId", "userId", "roleId")
    VALUES (v_team_id, v_user_id, c_role_tenant_admin);
  END IF;

  -- Section 2: environment (+ configPublic + envLabel), tenant, ODS instances, ownership
  SELECT id INTO v_sbenv_id FROM sb_environment WHERE name = v_env_name;
  IF v_sbenv_id IS NULL THEN
    INSERT INTO sb_environment (name, "envLabel", "configPublic", "configPrivate")
    VALUES (v_env_name, v_env_label, v_config_public, v_config_private) RETURNING id INTO v_sbenv_id;
  ELSE
    UPDATE sb_environment SET "envLabel" = v_env_label, "configPublic" = v_config_public,
           "configPrivate" = COALESCE(v_config_private, "configPrivate") WHERE id = v_sbenv_id;
  END IF;

  SELECT id INTO v_tenant_id FROM edfi_tenant
   WHERE "sbEnvironmentId" = v_sbenv_id AND name = v_edfi_tenant;
  IF v_tenant_id IS NULL THEN
    INSERT INTO edfi_tenant ("sbEnvironmentId", name) VALUES (v_sbenv_id, v_edfi_tenant)
    RETURNING id INTO v_tenant_id;
  END IF;

  -- ODS instances + one education organization each (edit / extend this list)
  FOR r IN
    SELECT * FROM (VALUES
      ('EdFi_Ods_2026', 5, 255901, 'Institution #255901', 'I#255901'),
      ('EdFi_Ods_2027', 6, 255902, 'Institution #255902', 'I#255902')
    ) AS t(ods_db_name, ods_instance_id, edu_org_id, edorg_name, edorg_short)
  LOOP
    SELECT id INTO v_ods_id FROM ods WHERE "edfiTenantId" = v_tenant_id AND "dbName" = r.ods_db_name;
    IF v_ods_id IS NULL THEN
      INSERT INTO ods ("edfiTenantId", "sbEnvironmentId", "odsInstanceId", "odsInstanceName", "dbName")
      VALUES (v_tenant_id, v_sbenv_id, r.ods_instance_id, r.ods_db_name, r.ods_db_name)
      RETURNING id INTO v_ods_id;
    END IF;

    SELECT id INTO v_edorg_id FROM edorg
     WHERE "edfiTenantId" = v_tenant_id AND "odsId" = v_ods_id AND "educationOrganizationId" = r.edu_org_id;
    IF v_edorg_id IS NULL THEN
      INSERT INTO edorg ("odsId", "odsDbName", "odsInstanceId", "edfiTenantId", "sbEnvironmentId",
                         "parentId", "educationOrganizationId", "nameOfInstitution",
                         "shortNameOfInstitution", discriminator)
      VALUES (v_ods_id, r.ods_db_name, r.ods_instance_id, v_tenant_id, v_sbenv_id,
              NULL, r.edu_org_id, r.edorg_name, r.edorg_short, v_edorg_discriminator)
      RETURNING id INTO v_edorg_id;
      INSERT INTO edorg_closure ("id_ancestor", "id_descendant") VALUES (v_edorg_id, v_edorg_id);
    END IF;
  END LOOP;

  IF NOT EXISTS (SELECT 1 FROM ownership WHERE "teamId" = v_team_id AND "sbEnvironmentId" = v_sbenv_id) THEN
    INSERT INTO ownership ("teamId", "sbEnvironmentId", "roleId") VALUES (v_team_id, v_sbenv_id, c_role_full_ownership);
  END IF;

  IF NOT EXISTS (SELECT 1 FROM ownership WHERE "teamId" = v_team_id AND "edfiTenantId" = v_tenant_id) THEN
    INSERT INTO ownership ("teamId", "edfiTenantId", "roleId") VALUES (v_team_id, v_tenant_id, c_role_full_ownership);
  END IF;
END $$;
```

```sql
SET NOCOUNT ON;
SET XACT_ABORT ON;

-- ---- Edit these variables ----
DECLARE @username    nvarchar(256) = N'admin@example.org';  -- = ADMIN_USERNAME / OIDC subject
DECLARE @teamName    nvarchar(256) = N'Quick Start';
DECLARE @envName     nvarchar(256) = N'Ed-Fi ODS/API v7.3';
DECLARE @envLabel    nvarchar(256) = N'QuickStart';
DECLARE @edfiTenant  nvarchar(256) = N'default';
DECLARE @edorgDiscriminator varchar(128) = 'edfi.Other';
-- configPublic to inject (single-quoted; edit URLs / adminApiKey to match your site)
DECLARE @configPublic nvarchar(max) = N'{"startingBlocks":false,"odsApiMeta":{"version":"7.3","applicationName":"Ed-Fi Alliance ODS/API","informationalVersion":"7.3.2","suite":"3","build":"7.3.20305.0","dataModels":[{"name":"Ed-Fi","version":"6.1.0","informationalVersion":"The Ed-Fi Data Model 6.1"}],"urls":{"dependencies":"https://localhost/WebApi/metadata/data/v3/dependencies","openApiMetadata":"https://localhost/WebApi/metadata/","oauth":"https://localhost/WebApi/oauth/token","oauthTokenIntrospection":"https://localhost/WebApi/oauth/token_info","dataManagementApi":"https://localhost/WebApi/data/v3/","xsdMetadata":"https://localhost/WebApi/metadata/xsd","changeQueries":"https://localhost/WebApi/changeQueries/v1/","composites":"https://localhost/WebApi/composites/v1/"}},"adminApiUrl":"https://localhost/AdminApi","version":"v2","values":{"meta":{"envlabel":"Development","mode":"SingleTenant","domainName":"https://localhost/WebApi","adminApiUrl":"https://localhost/AdminApi","tenantManagementFunctionArn":"","tenantResourceTreeFunctionArn":"","odsManagementFunctionArn":"","edorgManagementFunctionArn":"","dataFreshnessFunctionArn":""},"adminApiUuid":"0f1e4a03-b5cd-4af4-a84a-5df7b21579e8","tenants":{"default":{"adminApiKey":""}}}}';
-- Optional: set the encrypted secret so Admin API calls can authenticate.
DECLARE @configPrivate nvarchar(max) = NULL;  -- e.g. N'{"tenants":{"default":{"adminApiSecret":"<secret>"}}}'
-- ODS instances + one education organization each (edit / extend this list)
DECLARE @odsSeed TABLE (rn int IDENTITY(1,1), dbName nvarchar(256), odsInstanceId int,
                        eduOrgId bigint, edorgName nvarchar(256), edorgShort nvarchar(256));
INSERT INTO @odsSeed (dbName, odsInstanceId, eduOrgId, edorgName, edorgShort) VALUES
  (N'EdFi_Ods_2026', 5, 255901, N'Institution #255901', N'I#255901'),
  (N'EdFi_Ods_2027', 6, 255902, N'Institution #255902', N'I#255902');
-- ---- Built-in role ids (seeded by migrations; do not change) ----
DECLARE @roleGlobalAdmin   int = 2;  -- "Global admin"   (UserGlobal)
DECLARE @roleTenantAdmin   int = 6;  -- "Tenant admin"   (UserTenant)
DECLARE @roleFullOwnership int = 5;  -- "Full ownership" (ResourceOwnership)
-- ----
DECLARE @userId int, @teamId int, @sbEnvId int, @tenantId int, @odsId int, @edorgRowId int;

BEGIN TRY
  BEGIN TRANSACTION;

  -- Section 1: user, team, membership
  SELECT @userId = [id] FROM [user] WHERE [username] = @username;
  IF @userId IS NULL
  BEGIN
    INSERT INTO [user] ([username], [roleId], [isActive], [userType])
    VALUES (@username, @roleGlobalAdmin, 1, 'human');
    SET @userId = SCOPE_IDENTITY();
  END
  ELSE
    UPDATE [user] SET [roleId] = ISNULL([roleId], @roleGlobalAdmin), [isActive] = 1 WHERE [id] = @userId;

  SELECT @teamId = [id] FROM [team] WHERE [name] = @teamName;
  IF @teamId IS NULL
  BEGIN
    INSERT INTO [team] ([name]) VALUES (@teamName);
    SET @teamId = SCOPE_IDENTITY();
  END

  IF NOT EXISTS (SELECT 1 FROM [user_team_membership] WHERE [teamId] = @teamId AND [userId] = @userId)
    INSERT INTO [user_team_membership] ([teamId], [userId], [roleId]) VALUES (@teamId, @userId, @roleTenantAdmin);

  -- Section 2: environment (+ configPublic + envLabel), tenant, ODS instances, ownership
  SELECT @sbEnvId = [id] FROM [sb_environment] WHERE [name] = @envName;
  IF @sbEnvId IS NULL
  BEGIN
    INSERT INTO [sb_environment] ([name], [envLabel], [configPublic], [configPrivate])
    VALUES (@envName, @envLabel, @configPublic, @configPrivate);
    SET @sbEnvId = SCOPE_IDENTITY();
  END
  ELSE
    UPDATE [sb_environment] SET [envLabel] = @envLabel, [configPublic] = @configPublic,
           [configPrivate] = ISNULL(@configPrivate, [configPrivate]) WHERE [id] = @sbEnvId;

  SELECT @tenantId = [id] FROM [edfi_tenant] WHERE [sbEnvironmentId] = @sbEnvId AND [name] = @edfiTenant;
  IF @tenantId IS NULL
  BEGIN
    INSERT INTO [edfi_tenant] ([sbEnvironmentId], [name]) VALUES (@sbEnvId, @edfiTenant);
    SET @tenantId = SCOPE_IDENTITY();
  END

  -- ODS instances + one education organization each (loop over @odsSeed)
  DECLARE @i int = 1, @n int = (SELECT COUNT(*) FROM @odsSeed);
  DECLARE @dbName nvarchar(256), @odsInstanceId int, @eduOrgId bigint,
          @edorgName nvarchar(256), @edorgShort nvarchar(256);
  WHILE @i <= @n
  BEGIN
    SELECT @dbName = dbName, @odsInstanceId = odsInstanceId, @eduOrgId = eduOrgId,
           @edorgName = edorgName, @edorgShort = edorgShort FROM @odsSeed WHERE rn = @i;

    SELECT @odsId = [id] FROM [ods] WHERE [edfiTenantId] = @tenantId AND [dbName] = @dbName;
    IF @odsId IS NULL
    BEGIN
      INSERT INTO [ods] ([edfiTenantId], [sbEnvironmentId], [odsInstanceId], [odsInstanceName], [dbName])
      VALUES (@tenantId, @sbEnvId, @odsInstanceId, @dbName, @dbName);
      SET @odsId = SCOPE_IDENTITY();
    END

    SELECT @edorgRowId = [id] FROM [edorg]
     WHERE [edfiTenantId] = @tenantId AND [odsId] = @odsId AND [educationOrganizationId] = @eduOrgId;
    IF @edorgRowId IS NULL
    BEGIN
      INSERT INTO [edorg] ([odsId], [odsDbName], [odsInstanceId], [edfiTenantId], [sbEnvironmentId],
                           [parentId], [educationOrganizationId], [nameOfInstitution],
                           [shortNameOfInstitution], [discriminator])
      VALUES (@odsId, @dbName, @odsInstanceId, @tenantId, @sbEnvId,
              NULL, @eduOrgId, @edorgName, @edorgShort, @edorgDiscriminator);
      SET @edorgRowId = SCOPE_IDENTITY();
      INSERT INTO [edorg_closure] ([id_ancestor], [id_descendant]) VALUES (@edorgRowId, @edorgRowId);
    END

    SET @i = @i + 1;
  END

  IF NOT EXISTS (SELECT 1 FROM [ownership] WHERE [teamId] = @teamId AND [sbEnvironmentId] = @sbEnvId)
    INSERT INTO [ownership] ([teamId], [sbEnvironmentId], [roleId]) VALUES (@teamId, @sbEnvId, @roleFullOwnership);

  IF NOT EXISTS (SELECT 1 FROM [ownership] WHERE [teamId] = @teamId AND [edfiTenantId] = @tenantId)
    INSERT INTO [ownership] ([teamId], [edfiTenantId], [roleId]) VALUES (@teamId, @tenantId, @roleFullOwnership);

  COMMIT TRANSACTION;
END TRY
BEGIN CATCH
  IF @@TRANCOUNT > 0 ROLLBACK TRANSACTION;
  THROW;
END CATCH;
```

## Step 2: Set up ODS instances in `EdFi_Admin`

For the ODS/API to actually serve data for each instance, the corresponding
records must exist in the **`EdFi_Admin`** database. Populate `dbo.OdsInstances`
(and `dbo.OdsInstanceContexts` for year-specific ODS) with the same names you
seeded in Step 1 (`EdFi_Ods_2026`, `EdFi_Ods_2027`), then **restart the
ODS/API**.

Follow
[Creating Applications (Admin API v2 Mode)](/reference/admin-app/configuration/global-administration-tasks#creating-applications-admin-api-v2-mode)
for the exact SQL.

## Step 3: Sign in and verify

1. Sign in to the Admin App with the administrator username from Step 1.
2. Open the **Environments** page — `Ed-Fi ODS/API v7.3` is listed and owned by
   your team — and open it.
3. The **ODS instances** page lists `EdFi_Ods_2026` and `EdFi_Ods_2027`, and the
   **Ed-Orgs** page lists `Institution #255901` and `Institution #255902` (all
   read the local rows seeded in Step 1).
4. The **Applications** page calls the Admin API live; it works once
   `configPrivate` holds the tenant secret (and `dbo.OdsInstances` is
   populated).

ownership, ODS instances, Ed-Orgs, tenant, plus the team and its membership),
