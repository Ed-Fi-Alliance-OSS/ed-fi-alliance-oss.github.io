The Ed-Fi Alliance is excited to announce the release of **Data Standard v6.1**, **ODS/API v7.3.2**, **Ed-Fi OneRoster Service**, and **MetaEd v4.7**. These updates reflect continued collaboration with the Ed-Fi community and focus on advancing interoperability, modernizing the platform, and improving support for priority use cases.

<!-- truncate -->

## Data Standard v6.1

Ed-Fi Data Standard v6.1 introduces **non-breaking enhancements** designed to address key community needs. Key additions include an early-access Special Education data model to support richer IEP and IDEA event tracking, explicit tracking of an education organization's responsibility for a student, enhanced address representations, and improvements to documentation and tooling.

[What's New in Data Standard v6.1](https://docs.ed-fi.org/reference/data-exchange/data-standard/whats-new/whats-new-v61)

## ODS/API v7.3.2

ODS/API v7.3.2 aligns the platform with Data Standard v6.1 while continuing to support Data Standard 4.x and 5.x. It includes a **platform upgrade to .NET 10** ahead of the .NET 8 end-of-support deadline in November 2026 and introduces **OneRoster Service integration** to enable OneRoster API support within the Ed-Fi ecosystem.

[What's New in ODS/API 7.3.2](https://docs.ed-fi.org/reference/ods-api/whats-new/whats-new-in-this-release)

## Ed-Fi OneRoster Service

The Ed-Fi OneRoster Service is now available, implementing the **1EdTech® OneRoster v1.2 API**. It provides a standardized approach for exchanging roster data and supports broader interoperability across instructional and operational systems. ODS/API v7.3.2 or later is required to use this service.

[Ed-Fi OneRoster Documentation](https://docs.ed-fi.org/reference/oneroster/)

## MetaEd v4.7

MetaEd v4.7 includes tooling updates and fixes to support the latest Data Standard and platform releases, helping implementers stay aligned as they extend and customize Ed-Fi solutions.

[MetaEd Release Notes](https://docs.ed-fi.org/reference/metaed/releases/4.7.0)
