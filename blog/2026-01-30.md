## What's Included

### Ed-Fi OneRoster Release

The upcoming Ed‚ÄëFi OneRoster API enables agencies and vendors to serve a
OneRoster 1.2 endpoint using the data they already maintain within an Ed‚ÄëFi ODS.
Key capabilities include:

- **Broad Compatibility:** Compatible with Ed‚ÄëFi Data Standard 4.0 and 5.2
- **Supporting data structures:** Includes required mappings and database views
  to serve OneRoster API
- **Infrastructure Flexibility:** Supports both PostgreSQL and MSSQL backends
- **Standardized Deployment:** Officially published Docker image on Docker Hub
  to simplify setup and reduce onboarding effort

### ODS/API 7.3.2 Enhancements

The next iteration of the ODS/API introduces deep integration for OneRoster
resources:

- **Enhanced Discoverability:** OneRoster resources are surfaced through
  standard API discovery and Swagger UI, making the API easy to explore and test
- **Unified Login:** Features a unified login experience and integrated Ed-Fi
  Authorization across both standard Ed-Fi and OneRoster APIs
- **Built-in Security Metadata:** OneRoster-related OAuth scopes will be built
  into the core security configuration
- **Add-on Architecture:** The OneRoster API is surfaced as a seamless add-on
  feature, allowing for easier management and access
